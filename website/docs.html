Introduction

MINT — Meeting Intelligence Notes & Transcription — is a macOS desktop app that captures your microphone and your system audio simultaneously, transcribes both streams in real time, and turns the result into structured Markdown notes.

The app is local-first. Meetings are written directly to a folder on your machine — there is no MINT cloud, no account, no sync service. You choose which transcription and note providers to use, including fully offline options.

  • Dual audio capture — mic and system audio as separate, labeled streams.
  • Real-time transcription — Deepgram nova-2 over WebSocket, or local Whisper.
  • Local Markdown — every meeting is a folder of plain files you own.
  • Optional offline mode — Whisper plus Ollama for a fully local pipeline.

Requirements

  • macOS on Apple Silicon (M1 or newer).
  • Node.js 20 or newer.
  • Git.
  • A Deepgram API key — the free tier is enough to start.
  • Optional: an OpenAI API key (any chat model — defaults to GPT-4o), or a local Ollama install.
macOS only MINT is intentionally macOS-only. System audio capture relies on AudioTee, which is macOS-specific. Windows and Linux are not supported and are not on the roadmap.

Install

MINT ships as source. Clone the repo, install dependencies, and build the DMG locally. 

git clone https://github.com/ChrisWoo0443/MINT.git
cd MINT
npm install
npm run build:mac

When the build completes, open dist/ and drag the MINT.dmg into your Applications folder. Eject, launch from Spotlight, and you’re done.

Updating

To pick up a new release, pull and rebuild: 

git pull
npm run build:mac

The in-app update checker will also notify you when a new GitHub release is published.

First launch

On first launch, macOS will ask for two permissions. Both are required for the app to function:

  • Microphone — captures your voice.
  • Screen Recording — required by AudioTee to capture system audio (the other side of the call).

After granting permissions, the onboarding flow asks you to pick an input device and a storage folder. The default storage folder is ~/Documents/MINT.

Tip If you deny Screen Recording by accident, re-enable it under System Settings → Privacy & Security → Screen Recording and restart the app.

API keys

API keys are entered in the app’s Settings panel, not via a .env file. They’re persisted in Electron’s userData directory and never leave your machine except as outbound calls to the provider you chose.

  • Deepgram API key — for streaming cloud transcription.
  • OpenAI API key — for note generation. Once entered, MINT lists every chat model your key can access in a dropdown (defaults to gpt-4o).
  • Ollama URL — typically http://localhost:11434 if you run it on the same machine.

You only need to configure the providers you actually use. If you pick Whisper plus Ollama, no cloud key is required.

Providers

MINT separates the two AI tasks — transcription and note generation — so you can mix and match. Pick whichever combination fits the meeting.

Provider Role Latency Cost Privacy
Deepgram Transcription Real-time Pay-as-you-go Audio leaves device
Whisper (local) Transcription Near real-time on Apple Silicon Free Fully local
OpenAI Notes (any chat model) Seconds Pay-as-you-go Transcript leaves device
Ollama Notes Depends on model Free Fully local

Switch providers any time from Settings. The choice is per-app, not per-meeting.

Storage

By default, MINT writes meetings to ~/Documents/MINT. You can change this from Settings — point it at iCloud Drive, Dropbox, an external SSD, or any folder you control.

Each meeting is a directory:

~/Documents/MINT/
├── tags.json
└── 2026-05-08-team-sync/
    ├── metadata.json
    ├── transcript.md
    └── notes.md           # written when you generate notes

The format is intentionally portable. Open a meeting in Obsidian, Bear, VS Code, or just cat it. Tag definitions live in a single tags.json at the storage root.

Recording

Hit the record button in the top bar. MINT opens two audio streams simultaneously:

  • Microphone via the Web Audio API, labeled with your display name.
  • System audio via AudioTee, labeled Meeting Users.

Both are downsampled to 16 kHz linear16 PCM in the renderer, then forwarded to the main process over IPC. From there they fan out to whichever transcription provider you configured.

Stop the recording with the same button or ⌘ + .. MINT finalizes transcript.md and updates metadata.json on stop.

Live transcript

The recording view streams partial and finalized lines as they arrive, with speaker labels. You can pop out a compact floating overlay window — useful if you’re screen-sharing and want the transcript visible without taking focus from the meeting.

Past meetings are searchable from the dashboard. Search runs across transcripts, not just titles.

Notes generation

Notes are generated on demand, not automatically — MINT never sends your transcript anywhere until you click the Generate notes button.

The output is a structured Markdown file with three sections:

  • Summary — what the meeting was about.
  • Decisions — concrete things that were agreed.
  • Action items — owner and task, when extractable.

Regenerate as often as you want. Notes are saved as notes.md alongside the transcript.

Choosing a model

In Settings → Notes Generation, after entering your OpenAI API key MINT calls /v1/models and populates a dropdown of every chat model your key can access. Pick whichever you prefer — the default is gpt-4o.

Models in the gpt-4o, gpt-4.1, gpt-5, and o1/o3 families support OpenAI's strict structured-output mode and produce the most reliable JSON. Older models fall back to plain JSON mode automatically. Ollama users still pick from their locally installed models the same way.

Organization

The dashboard auto-sorts meetings into time-based sections — Today, Yesterday, This Week, Earlier. Drag a card between sections to override the automatic placement.

Color tags are defined once in tags.json and applied per meeting from the card. Tags persist as plain JSON, so they survive backups, syncs, and editor switches.

Architecture

MINT is a standard Electron app split across three processes — main, preload, and renderer. The renderer is treated as untrusted: every IPC payload is validated by a Zod schema in src/main/ipc-schemas.ts before reaching service code. 

src/
├── main/                       # Node.js — orchestration
│   ├── index.ts                # App entry, window + tray
│   ├── ipc-handlers.ts         # Recording lifecycle
│   ├── ipc-schemas.ts          # Zod validation
│   └── services/
│       ├── audiotee.ts         # System audio (macOS)
│       ├── deepgram.ts         # Streaming WebSocket
│       ├── transcription.ts    # Provider abstraction
│       ├── local-whisper.ts    # Offline transcription
│       ├── openai.ts           # OpenAI / Ollama notes
│       └── local-storage.ts    # Markdown + JSON
├── preload/                    # contextBridge — typed API
└── renderer/                   # React 19 + TypeScript

The audio pipeline: renderer captures both streams → downsamples to 16 kHz linear16 PCM → forwards over IPC (audio:chunk:mic, audio:chunk:system) → main fans out to one transcription stream per channel → results flow back to the renderer for display and persistence.

Contributing

Contributions are welcome. The full guide lives at CONTRIBUTING.md on GitHub. The short version:

  • Follow TDD — write the failing test first, then the implementation.
  • Keep commits atomic and descriptive.
  • Run npm run typecheck, npm test, and npm run lint before pushing.
  • macOS Apple Silicon is the only supported target. Don’t add platform shims for Windows or Linux.

Troubleshooting

Symptom Fix
System audio is silent in transcripts Grant Screen Recording permission in System Settings, then restart MINT.
Mic transcript is silent Check the input device in Settings and confirm macOS hasn’t muted MINT under Privacy & Security → Microphone.
Gatekeeper warns about an unidentified developer You build MINT yourself, so this should be rare. If it appears, see Apple’s docs on opening apps from unidentified developers.
Whisper model download is slow First run only. Models are cached locally after the initial download.
Transcription drops out mid-meeting The app auto-reconnects and surfaces a degraded-mode banner. If it persists, verify your Deepgram key and network.
Notes generation fails Confirm your OpenAI key has credit, or that Ollama is running and reachable at the configured URL.

Privacy

Audio and transcripts never leave your machine unless you have explicitly chosen a cloud provider. With Deepgram selected, audio is streamed to Deepgram for transcription. With OpenAI selected, the transcript is sent to OpenAI when you press Generate notes. Nothing else is uploaded.

For a fully offline pipeline, choose Whisper for transcription and Ollama for notes. In that configuration MINT makes no network calls related to your meetings.

Your meetings are files in a folder you own. That’s the whole point.
